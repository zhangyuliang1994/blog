# 4.1 🌐 网站类型

# 网站类型

网站是互联网上的一种服务形式，它通过浏览器访问，可以展示信息、提供服务、进行交易等。网站由多个网页组成，每个网页都有自己的URL，通过超链接相互连接。网站通常使用HTML、CSS和JavaScript等技术开发，并托管在服务器上。

## 传统网站开发技术栈

### 1. 静态网站（Static Site）

| 类别 | 说明 |
| --- | --- |
| **技术栈** | **HTML + CSS + JavaScript** |
| **特点** | 内容固定不变，适合个人简历、产品手册等场景。每次访问都返回相同的内容 |
| **优点** | 速度快，因为服务器直接返回静态文件 |
| **缺点** | 无法动态交互，不支持用户登录、实时数据等功能。内容更新需手动修改代码，不适合频繁更新的场景。 |
| **案例** | • 个人博客 • 公司官网 • 产品介绍页 |

### 2. 动态网站（Dynamic Site）

| 类别 | 说明 |
| --- | --- |
| **技术栈** | 后端语言（如 Node.js、Python、PHP）+ 数据库 + 前端框架 |
| **特点** | 内容动态生成，适合电商、社交平台等场景。每次访问时，服务器会根据用户请求实时生成内容 |
| **优点** | 支持丰富的用户交互功能，包括登录、评论、购物车等。内容可以实时更新，直接从数据库读取最新数据。 |
| **缺点** | 配置复杂，需要搭建后端、数据库和服务器环境 |
| **案例** | • 社交平台：微博、Facebook • 电商网站：淘宝、Amazon • 内容管理：WordPress |

## 总结：如何判断是否需要后端？

| **问题** | **不需要后端** | **需要后端** |
| --- | --- | --- |
| 用户需要登录/注册吗？ | ❌ 不需要 | ✅ 需要 |
| 内容是否实时更新（如库存）？ | ❌ 不需要 | ✅ 需要 |
| 需要保存用户数据吗？ | ❌ 不需要 | ✅ 需要 |
| 用户能提交表单或交互吗？ | ❌ 只能提交到第三方服务 | ✅ 需要自己处理数据 |

---

### 一句话记住：

- **不需要后端**：内容固定，用户只能看，不能提交数据。
- **需要后端**：用户要登录、提交内容，或数据需要实时更新。
- **取巧方案**：用第三方服务（如评论、支付）代替自己写后端。
