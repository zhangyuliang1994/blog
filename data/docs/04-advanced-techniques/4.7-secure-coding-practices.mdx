# 4.7 ⚡ Next.js 开发指南

# 认识 Next.js

Next.js 是一个现代化的 React 全栈开发框架，它最大的优势在于**统一了前后端开发**：只需掌握 JavaScript/React 一种语言，就能完整开发网站。框架内置了路由、API、数据库连接等功能，开箱即用无需配置。你可以专注写业务代码，框架自动处理性能优化，前后端代码在同一项目让调试更轻松。最后它还支持**一键部署到 Vercel**等平台，彻底告别复杂的服务器配置。

## 传统网站开发的痛点

为了更清晰理解为什么要用 Next.js，我们先对比传统开发网站的技术栈（静态网站和动态网站），再总结 Next.js 的优势。

| 痛点 | 说明 |
| --- | --- |
| **技术栈复杂** | 需要同时掌握前端(HTML/CSS/JS)和后端(PHP/Python等)多种语言 |
| **环境配置繁琐** | 要配置数据库、服务器、域名、SSL证书等，对新手不友好 |
| **开发效率低** | 一个人要写前端界面、后端接口、数据库操作，任务量大 |
| **维护成本高** | 出问题时要排查前端/后端/数据库，调试困难 |
| **部署复杂** | 需要购买服务器、配置环境、部署代码，容易出错 |

## Next.js 的优势

1. **全栈开发**：只需掌握 JavaScript/React，就能同时开发前端界面和后端接口，你不需要单独学习后端框架，就能实现前后端一体化开发。API 路由：在 pages/api 文件夹下写 JavaScript/TypeScript 文件，就能快速创建后端 API（比如处理表单提交、连接数据库）。
2. **零配置**：内置路由、API、数据库连接等功能，开箱即用。框架自动优化性能，让你专注写业务代码。
3. **简单部署**：Next.js 的母公司 Vercel 提供免费托管服务，无需复杂的服务器配置。前后端代码在同一项目，关联 GitHub 后点几下就能上线网站。调试也更轻松。支持多种托管平台：也可以部署到 Netlify、AWS 等平台。
4. **SEO友好度**：传统的前端框架（如纯 React）依赖浏览器渲染页面，但搜索引擎可能抓取不到内容。Next.js 通过预渲染生成完整的 HTML，让搜索引擎更容易理解你的网站内容，提高排名。
